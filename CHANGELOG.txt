1. CORRECTIONS DE BUGS CRITIQUES

[BUG] TEW.DICE.roll(range) — off-by-one
  Fichier : src/plugins/base/dice/Game_Interpreter.ts
  Problème : Math.floor(Math.random() * (range - 1)) + 1
             → un d10 ne pouvait jamais sortir 10, un d100 jamais 100.
  Fix      : Math.floor(Math.random() * range) + 1

  Fichier : src/plugins/base/dice/Game_Interpreter.ts (ligne ~133)
  Problème : successRollNpc = true quand rollNPC >= 96
             → un fumble ennemi était considéré comme une réussite.
  Fix      : successRollNpc = false

2. NOUVEAU SYSTÈME — CONDITIONS (WFRP4e p.167)
--------------------------------------------------------------------------------

Fichiers créés :
  src/plugins/_types/condition.ts
  src/plugins/constants/TEW_Conditions.ts
  src/plugins/combat/game/TEW_ConditionsSystem.ts

10 conditions implémentées avec piles (stacks) :
  ABLAZE, BLEEDING, BROKEN, ENTANGLED, FATIGUED,
  POISONED, PRONE, STUNNED, SURPRISED, UNCONSCIOUS

Chaque condition a :
  - Un nombre de piles (certaines stackables à l'infini)
  - Des effets par pile (dégâts/tour, blocage d'action, blocage de mouvement, fuite)
  - Un retrait automatique optionnel au début du tour (ex: SURPRISED)

API sur tout battler :
  battler.addCondition("BLEEDING", 2)
  battler.hasCondition("STUNNED")
  battler.conditionStacks("FATIGUED")
  battler.removeCondition("BROKEN", 1)
  battler.applyConditionsTurnStart()     → applique dégâts/tour, retire les temporaires
  battler.conditionTestModifier()         → somme des malus de test actifs
  battler.isBlockedByCondition()
  battler.isMovementBlockedByCondition()
  battler.mustFleeDueToCondition()

3. NOUVEAU SYSTÈME — BLESSURES CRITIQUES (WFRP4e p.159-163)
--------------------------------------------------------------------------------

Fichiers créés :
  src/plugins/_types/criticalInjury.ts
  src/plugins/constants/TEW_CriticalInjuries.ts
  src/plugins/combat/game/TEW_CriticalInjuriesSystem.ts

Tables complètes par localisation (40 entrées au total) :
  TÊTE  : 10 entrées (Dazed → Skull Caved In)
  TORSE : 9 entrées  (Winded → Mortal Wound)
  BRAS  : 7 entrées  (Bruised Arm → Arm Severed)
  JAMBES: 7 entrées  (Bruised Leg → Leg Severed)

Mécanique :
  - Déclenchée quand les dégâts réduisent la cible à 0 Wounds
  - Jet = d100 + excès de dégâts
  - Localisation = chiffre des unités du jet d'attaque (règle WFRP4e officielle)
    1-2 → Tête | 3-4 → Bras | 5-9 → Torse | 0 → Jambes
  - Chaque blessure applique des conditions, réduit des stats, peut être fatale

API :
  TEW.COMBAT.resolveCriticalInjury(target, attackRoll, excessDamage)
  TEW.COMBAT.healCriticalInjury(target, injuryId)
  battler.criticalInjuries()
  battler.criticalInjuryTestModifier()

4. NOUVEAU SYSTÈME — CORRUPTION & MUTATIONS (WFRP4e p.180-185)
--------------------------------------------------------------------------------

Fichiers créés :
  src/plugins/_types/corruption.ts
  src/plugins/constants/TEW_Corruption.ts
  src/plugins/combat/game/TEW_CorruptionSystem.ts

Deux types de corruption :
  - Physique → risque de Mutations (10 entrées : Warped Limb, Extra Eye, Horns...)
  - Mentale  → risque de Désordres (10 entrées : Paranoia, Phobia, Voices...)

Mécanique :
  - Seuil = bonus TOUG + bonus VOL
  - Dépassement → test d'Endurance Difficile
  - Échec → tirage aléatoire sur la table correspondante
  - Mutations/Désordres modifient les stats de façon permanente
  - Certains donnent des avantages (Iron Skin, Beast Eyes...)

API sur les acteurs :
  actor.gainPhysicalCorruption(2, "WARPSTONE")
  actor.gainMentalCorruption(1, "FORBIDDEN_TOME")
  actor.corruptionThreshold()
  actor.mutations()           → string[] des mutations actives
  actor.disorders()           → string[] des désordres actifs
  actor.corruptionStatModifier("FELW")
  actor.healPhysicalCorruption(1)
  actor.removeMutation("MUTATION_SCALY_SKIN")

5. NOUVEAU SYSTÈME — PSYCHOLOGIE : PEUR, TERREUR, FRÉNÉSIE (WFRP4e p.190)
--------------------------------------------------------------------------------

Fichier créé :
  src/plugins/combat/game/TEW_PsychologySystem.ts

PEUR :
  - Test de Cool (= WILL)
  - Échec → condition BROKEN × rating Fear
  - Succès → immunité à cette source pour tout l'affrontement

TERREUR :
  - Test de Cool avec malus −rating × 10
  - Échec → BROKEN + tirage d'un Désordre mental
  - Succès → immunité + 1 point de Corruption Mentale (même les braves sont marqués)

FRÉNÉSIE :
  - Immunité à Peur et Terreur
  - +1 SL sur toutes les attaques
  - Se termine quand plus d'ennemi en vue → gain de FATIGUED 1
  - Nettoyage automatique en fin de combat (hook sur BattleManager.endBattle)

API :
  battler.resolveFearTest(2, "BEASTMAN_CHIEFTAIN")
  battler.resolveTerrorTest(3, "CHAOS_SPAWN")
  battler.enterFrenzy()
  battler.exitFrenzy()
  battler.isInFrenzy()
